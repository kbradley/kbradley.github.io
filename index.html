About

Bricks

intuitive responsive CSS layout system

A keyword-based contextual CSS layout library that works the way you expect it to!

Create typical layout patterns using a combination of simple key word classes. Add responsive suffixes to target by view. Works together with any type of frontend UI component framework or none.

Use simple, easy to remember key word classes together with modifier classes to intuitively create common layout patterns. Classes can work differently based on contect. (For example, "center" will either center text or groups of elements based on the type of element it's applied to.) Allows non css savvy devs to apply key words that do what they want and expect even if they don't understand the underlying differences in how it affects various content. Makes responsive layouts simple through the use of responsive class suffixes Supports modern browsers, IE11, and Edge

What makes Bricks special? Bricks is focused only on abstract lay out patterns and simple reusable utility styles Classes are context-based so may work differently depending on the type of element they are used on Designed to be easy to use by non-CSS experts

Column and Grid Layouts

Default columns

Brick are all equal width by default. They will not wrap unless forced to. (IE: if they run out of room to fit on one line)

The content of default columns may stretch them wider. Set explicit widths to prevent this.

Equal width columns / grid

Bricks with explicitly set equal widths. They will wrap if there are more than will fit.

Setting widths explicitly prevents content from stretching bricks and makes them maintain their set width if they wrap to the next line.

You can add a width style either to a brick-group or to individual bricks. Widths applied to brick-groups affect all the child bricks inside.

Responsive

Most css classes in Bricks.css work together with optional responsive suffixes which can be used to target specific responsive views.

To target a class only to a particular view add one of 3 suffixes to the end. To apply a style to all views, just use the non-suffixed class name.

To target multiple views, add multiple classes with responsive suffixes.

Responsive Suffixes:

  • @mobile
  • @tablet
  • @desktop

Example usage:

Lorem ipsum dolor sit amet
markup 
<div class="center@mobile right@tablet right@desktop">
 Lorem ipsum dolor sit amet
</div>

Spacing

Margins, padding, and gutters come in a standardized range of sizes:

Spacing Sizes:

  • xs = 12px (40px * .3) Extra Small
  • sm = 20px (40px * .5) Small
  • ms = 30px (40px * .75) Medium Small
  • md / default = 40px (all other spacing sizes are based off this default)
  • lg = 60px (40px * 1.5) Large
  • xl = 80px (40px * 2) Extra Large NOTE: spacing sizes actually use rem not px.
